# 简化:代码审查和清理

审查所有更改的文件的可重用性、质量和效率。修复发现的任何问题。

## 阶段 1:识别更改

运行 \`git diff\`(或 \`git diff HEAD\` 如果有暂存的更改)以查看更改了什么。如果没有 git 更改,审查用户提到的或你在本次对话中较早编辑的最近修改的文件。

## 阶段 2:并行启动三个审查代理

使用 ${AGENT_TOOL_NAME} 工具在单个消息中并发启动所有三个代理。向每个代理传递完整的 diff,以便它具有完整的上下文。

### 代理 1:代码重用审查

对于每个更改:

1. **搜索现有的实用程序和助手**,可以替换新编写的代码。在代码库的其他地方寻找类似的模式 — 常见位置是实用程序目录、共享模块以及更改文件附近的文件。
2. **标记任何重复现有功能的新函数。** 建议改用现有函数。
3. **标记可以使用现有实用程序的内联逻辑** — 手动字符串操作、手动路径处理、自定义环境检查、临时类型守卫和类似模式是常见候选。

### 代理 2:代码质量审查

审查相同的更改是否有hack模式:

1. **冗余状态**:重复现有状态的状态、可以派生的缓存值、可能是直接调用的观察者/效果
2. **参数膨胀**:向函数添加新参数而不是泛化或重构现有参数
3. **略有变化的复制粘贴**:应该通过共享抽象统一的近乎重复的代码块
4. **泄漏抽象**:暴露应该封装的内部细节,或破坏现有的抽象边界
5. **字符串类型代码**:在代码库中已存在常量、枚举(字符串联合)或品牌类型的地方使用原始字符串
6. **不必要的 JSX 嵌套**:不添加布局值的包装器 Box/元素 — 检查内部组件 props(flexShrink、alignItems 等)是否已提供所需的行为

### 代理 3:效率审查

审查相同更改的效率:

1. **不必要的工作**:冗余计算、重复文件读取、重复的网络/API 调用、N+1 模式
2. **错过的并发**:可以并行运行的独立操作顺序运行
3. **热路径膨胀**:向启动或每个请求/每个渲染热路径添加了新的阻塞工作
4. **经常性的无操作更新**:轮询循环、间隔或事件处理程序中的状态/存储更新无条件触发 — 添加更改检测保护,以便当下游使用者在没有任何更改时不会被通知。还有:如果包装函数接受 updater/reducer 回调,验证它遵守相同的引用返回(或任何"无更改"信号) — 否则调用者的早期返回无操作会被静默击败
5. **不必要的存在性检查**:在操作之前预先检查文件/资源存在性(TOCTOU 反模式) — 直接操作并处理错误
6. **内存**:无界数据结构、缺少清理、事件监听器泄漏
7. **过于广泛的操作**:读取整个文件时只需要一部分,加载所有项目时过滤一个

## 阶段 3:修复问题

等待所有三个代理完成。汇总他们的发现并直接修复每个问题。如果发现是误报或不值得解决,请记录并继续 — 不要与发现争论,只需跳过它。

完成后,简要总结修复了什么(或确认代码已经是干净的)。
