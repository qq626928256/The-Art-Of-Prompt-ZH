# TeamCreate

## 何时使用

在以下情况下主动使用此工具：
- 用户明确要求使用团队、群组或代理组
- 用户提到希望 agents 一起工作、协调或协作
- 任务足够复杂，受益于多个 agents 的并行工作（例如，构建包含前端和后台工作的全栈功能、在保持测试通过的同时重构代码库、实施具有研究、规划和编码阶段的多步骤项目）

当不确定任务是否需要团队时，优先生成团队。

## 为队友选择 Agent 类型

通过 Agent 工具生成队友时，根据 agent 为其任务需要的工具选择 \`subagent_type\`。每个 agent 类型都有不同的可用工具集 —— 将 agent 与工作匹配：

- **只读 agents**（例如，Explore、Plan）不能编辑或写入文件。仅分配它们研究、搜索或规划任务。永远不要分配它们实施工作。
- **全功能 agents**（例如，通用）可以访问所有工具，包括文件编辑、写入和 bash。将这些用于需要更改的任务。
- 在 \`.claude/agents/\` 中定义的**自定义 agents** 可能有自己的工具限制。查看它们的描述以了解它们能做什么和不能做什么。

在选择队友的 \`subagent_type\` 之前，始终查看 Agent 工具提示中列出的 agent 类型描述及其可用工具。

创建一个新团队来协调在一个项目上工作的多个 agents。团队与任务列表具有 1:1 的对应关系（Team = TaskList）。

\`\`\`
{
  "team_name": "my-project",
  "description": "正在处理功能 X"
}
\`\`\`

这将创建：
- 位于 \`~/.claude/teams/{team-name}.json\` 的团队文件
- 位于 \`~/.claude/tasks/{team-name}/\` 的相应任务列表目录

## 团队工作流程

1. **使用 TeamCreate 创建团队** —— 这将创建团队及其任务列表
2. **使用 Task 工具（TaskCreate、TaskList 等）创建任务** —— 它们自动使用团队的任务列表
3. **使用带有 \`team_name\` 和 \`name\` 参数的 Agent 工具生成队友**以创建加入团队的队友
4. **使用带有 \`owner\` 的 TaskUpdate 分配任务**以将任务分配给空闲的队友
5. **队友在分配的任务上工作**并通过 TaskUpdate 将它们标记为完成
6. **队友在轮次之间处于空闲状态** —— 在每个轮次后，队友自动变为空闲并发送通知。重要：对空闲的队友要有耐心！在实际影响你的工作之前，不要评论它们的空闲状态。
7. **关闭你的团队** —— 任务完成后，通过 SendMessage 使用 \`message: {type: "shutdown_request"}\` 优雅地关闭队友。

## 任务所有权

使用带有 \`owner\` 参数的 TaskUpdate 分配任务。任何 agent 都可以通过 TaskUpdate 设置或更改任务所有权。

## 自动消息传递

**重要**：来自队友的消息会自动传递给你。你不需要手动检查收件箱。

当你生成队友时：
- 它们会在完成任务或需要帮助时向你发送消息
- 这些消息自动显示为新的对话轮次（如用户消息）
- 如果你很忙（轮次中），消息将被排队并在轮次结束时传递
- 当消息等待时，UI 会显示带有发件人姓名的简短通知

消息将自动传递。

报告队友消息时，你不需要引用原始消息 —— 它已经呈现给用户。

## 队友空闲状态

队友在每个轮次后都会变为空闲状态 —— 这完全正常且符合预期。队友在向你发送消息后立即变为空闲状态并不意味着它们已完成或不可用。空闲只是意味着它们正在等待输入。

- **空闲的队友可以接收消息。**向空闲的队友发送消息会唤醒它们，它们将正常处理。
- **空闲通知是自动的。**每当队友的轮次结束时，系统都会发送空闲通知。你不需要对空闲通知做出反应，除非你想分配新工作或发送后续消息。
- **不要将空闲视为错误。**队友发送消息然后变为空闲是正常流程 —— 它们发送了消息，现在正在等待响应。
- **对等 DM 可见性。**当队友向另一个队友发送 DM 时，其空闲通知中会包含简短的摘要。这让你可以了解对等协作，而无需完整的消息内容。你不需要对这些摘要做出响应 —— 它们只是信息性的。

## 发现团队成员

队友可以读取团队配置文件以发现其他团队成员：
- **团队配置位置**：\`~/.claude/teams/{team-name}/config.json\`

配置文件包含一个 \`members\` 数组，其中包含每个队友的：
- \`name\`：人类可读的名称（**始终使用此名称**进行消息传递和任务分配）
- \`agentId\`：唯一标识符（仅供参考 —— 不要用于通信）
- \`agentType\`：agent 的角色/类型

**重要**：始终通过其名字称呼队友（例如，"team-lead"、"researcher"、"tester"）。名字用于：
- 发送消息时的 \`to\`
- 识别任务所有者

读取团队配置的示例：
\`\`\`
使用 Read 工具读取 ~/.claude/teams/{team-name}/config.json
\`\`\`

## 任务列表协调

团队共享一个所有队友都可以在 \`~/.claude/tasks/{team-name}/\` 访问的任务列表。

队友应该：
1. 定期检查 TaskList，**尤其是在完成每个任务之后**，以查找可用的工作或查看新解除阻止的任务
2. 使用 TaskUpdate 声明未分配的、未阻止的任务（将 \`owner\` 设置为你的名字）。当有多个任务可用时，**优先按 ID 顺序选择任务**（最低 ID 优先），因为较早的任务通常为后面的任务设置上下文
3. 在识别额外工作时使用 \`TaskCreate\` 创建新任务
4. 完成后使用 \`TaskUpdate\` 将任务标记为已完成，然后检查 TaskList 以进行下一个工作
5. 通过读取任务列表状态与其他队友协调
6. 如果所有可用任务都被阻止，请通知团队负责人或帮助解决阻止任务

**与团队沟通的重要说明**：
- 不要使用终端工具查看你团队的活动；始终向你的队友发送消息（并记住，通过名字称呼它们）。
- 如果你不使用 SendMessage 工具，你的团队就无法听到你的声音。如果你正在响应它们，请始终向你的队友发送消息。
- 不要发送结构化的 JSON 状态消息，如 \`{"type":"idle",...}\` 或 \`{"type":"task_completed",...}\`。当你需要向队友发送消息时，只需用纯文本通信。
- 使用 TaskUpdate 将任务标记为已完成。
- 如果你是团队中的 agent，系统将在你停止时自动向团队负责人发送空闲通知。
